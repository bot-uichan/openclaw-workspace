# Nostr Search

Nostr検索タスクを実行する前に、まず `references/spec.md` を読んで要件を確定する。
「Relayからデータ取得方法の文書化」が目的なら `references/relay-data-acquisition.md` を優先して使う。

## 進め方

1. 検索目的を確認する（監視、調査、人物追跡、話題収集）。
2. 入力条件を確定する（キーワード、npub/nprofile、#tag、期間、言語）。
3. 出力形式を確定する（一覧、要約、JSON、アラート用短文）。
4. `references/spec.md` の未確定項目を埋める。
5. 実装フェーズに入るまでは、API呼び出し・スクリプト作成をしない。

## 出力ルール

- 検索対象・検索条件・除外条件を明示する。
- 取得件数上限と新着優先/関連度優先を明示する。
- 実装前は「未実装」「設計のみ」を明記する。

## 実装フェーズへの受け渡し

実装着手時は以下を必須入力として引き継ぐ。

- Relay/API候補
- 検索パラメータ仕様
- レート制限/リトライ方針
- エラー時フォールバック
- 最終出力フォーマット
